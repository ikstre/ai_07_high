# 2026-06-02 Report

## 작업 요약

- 사용자가 선택한 3번 시안인 `광고 제작 스튜디오` 방향으로 Streamlit UI를 수정했다.
- 기존 API 호출, 렌더링 로직, 광고 생성 로직은 유지하고 화면 구성과 스타일 중심으로 변경했다.
- 상단에 캠페인 브리프 히어로 영역을 추가해 상품명, 채널, 타깃, 가격, 광고 톤을 한눈에 보이게 했다.
- 4단계 진행 구조를 광고 제작 파이프라인 카드로 시각화했다.
- 광고 콘텐츠 단계의 입력 패널을 `광고 제작 스튜디오`로 바꾸고 SNS 카드, 상세 배너, 카피 세트 제작 흐름을 강조했다.
- 결과 영역을 `광고 제작 결과` 보드로 바꾸고 3D 씬, 광고 문구, 포스터 준비 상태 카드를 추가했다.
- 광고 카드 미리보기 스타일을 어두운 캠페인 카드 형태로 변경해 포스터/광고 결과물 느낌을 강화했다.
- 사이드바에 `화면 모드` 선택을 추가해 `기본 베이스`와 `다크 모드`를 전환할 수 있게 했다.
- 기본값은 `기본 베이스`로 설정했고, 두 모드는 같은 레이아웃을 공유하면서 색상 토큰만 다르게 적용되도록 구성했다.
- `기본 베이스`에서도 남아 있던 어두운 Streamlit 헤더와 JSON/code 표시 영역을 밝은 색상 토큰으로 보정했다.
- `기본 베이스`가 너무 순백으로 보여 피로감이 있는 문제를 줄이기 위해 기본 배경을 소프트 블루그레이 톤으로 조정했다.
- 기본 모드의 앱 배경, 사이드바, 카드 표면, 입력창, 코드 블록, 헤더 색상을 `#eef3f8` 계열 중심으로 재조정했다.
- 기본 계열을 스카이블루 방향으로 다시 잡고, 사이드바 비교용으로 `스카이블루 + 밝은 사이드`와 `스카이블루 + 어두운 사이드` 두 옵션을 추가했다.
- 기존 `다크 모드`는 유지해 총 3개 화면 모드를 선택할 수 있게 했다.
- 추가 비교 요청에 따라 `세이지 블루`와 `아이스 네이비 사이드` 테마를 추가했다.
- 기본 선택값은 추천안인 `아이스 네이비 사이드`로 변경했다.
- `아이스 네이비 사이드` 선택 시 사이드바 expander/입력/캡션 텍스트가 흐려지는 문제를 사이드바 전용 색상 오버라이드로 보정했다.
- textarea/input 경계가 배경에 묻혀 라인 위치가 잘 보이지 않는 문제를 해결하기 위해 기본/포커스 테두리와 focus ring을 강화했다.
- 상품명/판매가 입력 시 텍스트 커서(caret)가 잘 보이지 않는 문제를 `caret-color` 지정으로 보정했다.
- 기본/아이스 네이비 테마에서 selectbox 드롭다운 메뉴가 다크 배경으로 깨져 보이는 문제를 BaseWeb menu/popover 색상 오버라이드로 보정했다.
- 1단계 상품 정보 폼에서 드롭다운(`상품 유형`, `판매 채널`)을 오른쪽 보조 컬럼으로 이동하고 입력 패널 폭을 넓혔다.
- 상품명, 판매가, 타깃 고객, 핵심 특징은 왼쪽 메인 컬럼에 배치해 텍스트 입력 영역을 더 넓게 유지했다.
- 결과 캔버스 탭 순서를 `광고 카드` → `가상 셋업` → `업로드 모델` → `광고 결과물`로 변경했다.
- 기존 하단 광고 카드 미리보기 블록을 `광고 카드` 탭 안으로 이동해 같은 결과 레이어에서 먼저 확인되도록 했다.
- 3D 생성 전 JSON 설정값 안내는 Step 3(`가상 셋업`)일 때만 자세히 표시되도록 변경했다.
- selectbox 내부에 보이던 텍스트 커서형 표시를 숨기고, 오른쪽 드롭다운 아이콘은 Streamlit/BaseWeb 기본 SVG 아이콘을 다시 사용하도록 복원했다.
- 커스텀 chevron(`›`, `⌄`) 오버라이드는 기존 selectbox 구조와 충돌하므로 제거했다.
- selectbox 내부 input이 일반 input 스타일을 다시 물려받아 흰 커서/사각형처럼 보이는 문제를 해결하기 위해 selectbox input을 1px 투명 처리했다.
- selectbox 기본 SVG 아이콘 색상만 테마에 맞게 조정했다.
