## CSS 모듈 분리

- Streamlit 화면 모드 CSS 렌더링 함수를 `deskad_keyboard_demo/ui/styles.py`로 분리했다.
- `streamlit_app.py`는 CSS 문자열을 직접 들고 있지 않고 `render_ui_theme_styles(st.session_state.get("ui_theme_mode"))`를 호출하도록 변경했다.
- `ui/styles.py`는 `ui/theme.py`의 `theme_tokens()`를 사용하므로 색상 토큰과 CSS 적용 책임을 분리한 상태다.
- 이번 분리는 UI 코드 크기를 줄이는 목적이며, 화면 동작 자체는 변경하지 않았다.

## API/cache 모듈 분리

- Streamlit에서 반복 사용하던 API 호출 래퍼와 캐시 fetch 함수를 `deskad_keyboard_demo/ui/api_client.py`로 분리했다.
- 분리한 항목:
  - `api_get()`
  - `api_post()`
  - `fetch_binary_data_url()`
  - `fetch_text_asset()`
  - `fetch_security_config()`
  - `fetch_ai_providers()`
  - `fetch_desk_assets()`
  - `fetch_layout_ids()`
  - `fetch_reference_assets()`
  - `fetch_model_library()`
- API base URL과 `.env` 로딩도 `ui/api_client.py`로 옮겨서 `streamlit_app.py`가 API 환경 설정을 직접 들고 있지 않게 했다.
- `FALLBACK_ASSETS`, `DEFAULT_LAYOUT_FALLBACK`은 캐시 fetch 함수 내부에서만 쓰이므로 함께 이동했다.
- `streamlit_app.py`는 새 모듈에서 함수들을 import해서 기존 호출 흐름을 유지한다.
