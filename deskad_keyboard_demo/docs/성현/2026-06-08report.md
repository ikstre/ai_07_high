## 수정 내용
- 밝은 모드 기본 배경을 `#edf5fb`로 조정해 순백 화면의 눈부심을 줄였다.
- 사이드바 배경을 아이스 네이비 계열로 고정하고 제목, 라벨, 보조 텍스트 대비를 높였다.
- 사이드바 폭은 과하게 줄였을 때 메뉴 줄바꿈이 심해져 최종적으로 `280px`로 복구했다.
- 입력창, textarea, number input, select 계열의 배경, 글자색, 테두리, 커서색을 명시했다.
- focus 상태에 파란 outline을 추가해 현재 입력 위치가 보이도록 했다.
- select 드롭다운 메뉴의 최소 폭, 배경, hover 색, 그림자를 지정해 메뉴가 입력창과 겹쳐 답답해 보이는 문제를 줄였다.
- Streamlit expander 아이콘 폰트가 로드되지 않을 때 `keyboard_arrow_down/right` 텍스트가 그대로 보이는 문제를 CSS로 보정했다.
  - 실제 material icon 텍스트는 `font-size: 0`으로 숨겼다.
  - 닫힘 상태는 `›`, 열림 상태는 `⌄`를 `::before`로 표시했다.
- 일반 사용자 화면에서 내부 디버그 정보인 `현재 렌더링 payload` expander와 JSON 출력을 제거했다.
- 3D 결과가 없을 때 안내 카드와 `3D 데스크 셋업 생성` 버튼을 표시하도록 바꿨다.
- 이후 버튼 위치가 결과 카드 중앙에 떠 보여서, 빈 3D 카드에서는 버튼을 제거했다.
- `3D 데스크 셋업 생성` 버튼은 Step 3 내부에 이미 존재하므로 메인 결과/편집 레이아웃에서는 중복 노출하지 않도록 정리했다.
- `광고 포스터 / 이미지 작업`, `광고 카드 / 문구 후보` 출력 영역은 Step 4 전에는 보이지 않도록 조건부 렌더링으로 미뤘다.
- Step 1~3 상단 설명에서는 광고 포스터와 문구를 한 화면에 표시한다는 표현을 빼고, 현재 단계에 맞는 3D 셋업 중심 설명으로 바꿨다.
- `가상 데스크 셋업 결과`, 3D 셋업 미리보기, KB/Mon/Desk 메타 칩, `결과 새로고침` 버튼은 Step 3부터 보이도록 조정했다.
- Step 1~2의 왼쪽 결과 영역은 3D 정보 대신 현재 입력 단계 안내 카드만 표시하도록 바꿨다.
- Step 1~2에서는 실제 입력 작업이 우선이므로 `선택 편집` 컬럼을 왼쪽 넓은 영역으로 보내고 안내 카드는 오른쪽 보조 영역으로 배치했다.
- Step 3부터는 3D 결과 확인이 우선이므로 기존처럼 3D 결과 영역을 왼쪽 넓은 영역에 유지한다.
- Step 4 광고 콘텐츠의 `광고 톤`, `이미지 비율`, `포스터 템플릿` 드롭다운을 제거하고 클릭형 버튼/카드 선택 UI로 바꿨다.
- 포스터 템플릿은 썸네일 카드 안에서 바로 선택하도록 바꿔, 별도 selectbox와 썸네일이 중복으로 보이지 않게 정리했다.
- 선택된 버튼이 primary 스타일로 바뀔 때 pill 형태로 radius가 커지지 않도록 버튼 radius를 `8px`로 고정했다.
- 포스터 템플릿 카드의 별도 `선택/선택됨` 하단 버튼을 제거하고, 카드 제목 버튼을 눌러 선택하도록 바꿨다.
- 선택된 포스터 템플릿 버튼은 `선택됨` 텍스트를 붙이지 않고 색상만 바뀌도록 정리했다.
- 포스터 템플릿명이 길 때 버튼 안에서 2줄로 보이도록 버튼 텍스트 줄바꿈을 허용했다.
- Step 4 광고 콘텐츠 영역의 컬럼 비율을 조정해 포스터 템플릿 카드 영역을 조금 더 넓혔다.
- Step 4는 광고 설정이 주 작업이므로 전체 결과/편집 컬럼 비율을 `25:75`로 조정해 템플릿 카드가 너무 좁아지지 않게 했다.

## 사용자 친화 UI 재구성
- 사이드바를 운영/진단 패널 중심에서 `진행 요약`, `단계 이동`, `화면 모드` 중심으로 재구성했다.
- `API / 보안 상태`, `도면 데이터`, `렌더링 설정`, `광고 산출물`은 `고급 설정 / 진단` 아래에 기본 접힘 상태로 숨겼다.
- 결과 패널을 단계별 주 작업에 맞춰 재배치했다.
  - Step 1~2: 입력 폼을 왼쪽 넓은 영역에 배치하고, 안내 카드는 오른쪽 보조 영역으로 축소했다.
  - Step 3: 3D 셋업 결과, 메타데이터, 새로고침을 왼쪽 결과 영역에 유지했다.
  - Step 4: 왼쪽은 광고 카드/포스터 미리보기, 오른쪽은 광고 톤/비율/템플릿 선택과 생성 액션으로 분리했다.
- Step 4에서 포스터가 생성되면 왼쪽 미리보기 영역에 바로 표시되도록 하고, 다운로드와 이미지 작업 상세는 하단 상세 expander로 분리했다.
- 버튼 선택 UI는 색상, radius, 줄바꿈 규칙을 유지하면서 새 모듈 구조의 `ui/styles.py`에 반영했다.
- Step 3 진입 시 `selected_setup_item` 기본값이 없어 발생하던 session_state 오류를 막기 위해 기본 선택 항목을 `keyboard`로 추가했다.
- `선택 편집` 영역이 `3D 셋업` preview column 안에 잘못 들어가던 들여쓰기 문제를 수정했다.
- Step 3 화면에서 한 화면 밀도를 맞추기 위해 block padding, heading, paragraph, label, divider, vertical gap을 줄였다.
- 상단 작업 영역이 브라우저 상단에 붙어 보여 main block 상단 padding을 `3.55rem`으로 다시 늘렸다.
- 상단 단계 statusbar가 너무 위에 붙지 않도록 `.step-progress`의 위쪽 margin을 추가했다.
- 3D model viewer 높이는 과도하게 줄이면 결과 확인성이 떨어져 `540px`로 조정했다.
- Step 3의 고급 키보드 상세 옵션은 `고급 키보드 옵션` expander 안으로 접었다.
  - 기본 화면에는 키보드 모델, 배열, 하우징/키캡/포인트키 색상만 먼저 보인다.
  - 케이스 마감, 보강판 재질, 스위치 구조, stem, 키캡 프로파일, 마운트 방식, 내부 구조 렌더 노출은 고급 옵션에서 조정한다.
