## OpenAI 이미지 모델 입력 보정

기존 문제:

- `.env`에 `OPENAI_IMAGE_MODEL=GPT-image-1-mini`처럼 대소문자가 섞여 들어가면 OpenAI Image API 요청이 실패할 수 있었다.
- 화면에는 `openai_image · failed · 1024x1024` 정도만 보여서 실제 실패 원인을 바로 알기 어려웠다.

수정 내용:

- `deskad_keyboard_demo/backend/ai.py`
  - `settings.openai_image_model.strip().lower()`를 사용해 모델명을 요청 직전에 소문자로 정규화했다.
  - 예: `GPT-image-1-mini` → `gpt-image-1-mini`
  - OpenAI 이미지 생성 실패 시 `400 Bad Request`만 반환하지 않고, OpenAI 응답 본문 일부를 함께 남기도록 수정했다.

검증:

- 실제 비용이 발생하지 않도록 `_request_json`을 monkeypatch해서 요청 payload만 확인했다.
- 확인 결과 모델명이 `gpt-image-1-mini`로 전송되는 것을 확인했다.

## 광고 콘텐츠 화면 폭 확장

기존 문제:

- 결과 영역이 좁게 보여 1:1 포스터와 광고 카드 확인이 답답했다.
- 화면 라벨이 `INPUT PANEL / 620 x 500`, `RESULT CANVAS / 1000 x 1000`처럼 고정 크기처럼 보였다.

수정 내용:

- `deskad_keyboard_demo/streamlit_app.py`
  - 본문 최대 폭을 `1680px` 고정에서 `min(96vw, 1920px)`로 넓혔다.
  - 좌우 padding을 줄여 실제 사용 가능한 가로 공간을 늘렸다.
  - 사이드바 폭을 `300px`에서 `280px`로 줄였다.
  - 입력/결과 컬럼 비율을 `[0.62, 1.35]`에서 `[0.72, 1.85]`로 조정했다.
  - 라벨을 `INPUT PANEL / responsive`, `RESULT CANVAS / responsive`로 변경했다.

검증:

- 브라우저 새로고침 후 `INPUT PANEL / responsive`, `RESULT CANVAS / responsive`가 화면에 반영된 것을 확인했다.

## 광고 카드 미리보기 데이터 매핑 수정

기존 문제:

- 광고 카드 미리보기가 생성된 광고 문구를 충분히 반영하지 않고, 상품명/핵심 특징 위주로 표시됐다.
- 사용자가 보기에는 생성 문구가 카드에 잘못 들어가는 것처럼 보였다.

수정 내용:

- `deskad_keyboard_demo/streamlit_app.py`
  - 생성된 `copy_result`가 있으면 아래 필드를 광고 카드에 우선 반영하도록 수정했다.
    - `headline`
    - `subcopy`
    - `cta`
    - `copies`
  - 생성 결과가 없을 때만 기존 상품명/핵심 특징을 fallback으로 표시한다.
  - 카드 형태를 별도 `.ad-preview-card` 스타일로 구성해 문구 계층이 더 명확하게 보이도록 했다.

## 포스터 SVG 미리보기 반응형 처리

기존 문제:

- 포스터 SVG가 1000px 기준으로 iframe에 들어가 가로 스크롤이 생겼다.
- 1:1 포스터를 확인할 때 실제 결과 영역보다 좁고 불편하게 보였다.

수정 내용:

- `deskad_keyboard_demo/streamlit_app.py`
  - `responsive_svg_document()` helper를 추가했다.
  - SVG를 iframe 내부에서 `width: 100%; height: auto; max-width: 100%;`로 표시하도록 래핑했다.
  - 포스터 미리보기 높이를 `600`에서 `760`으로 늘렸다.
  - 기존 `scrolling=True`를 `scrolling=False`로 변경해 가로 스크롤을 줄였다.

## 검증 명령

python -m compileall backend streamlit_app.py

결과:

- 비용 없는 monkeypatch 테스트로 OpenAI 이미지 모델명 정규화 확인

## 적용 시 주의사항

- Streamlit UI 변경은 브라우저 새로고침으로 반영된다.
- `backend/ai.py` 변경은 실행 중인 uvicorn 서버를 재시작해야 반영된다.
- 현재 `.env`의 이미지 모델 값은 `OPENAI_IMAGE_MODEL=gpt-image-1-mini` 형태를 권장한다.

## 남은 확인 사항

- 광고 포스터 미리보기는 이전보다 넓게 표시되지만, 1:1 포스터 전체 높이를 한 화면에서 모두 보여주지는 못해 하단 일부가 스크롤 영역 아래로 내려갈 수 있다.

- 커밋 시 `__pycache__`, `tmp_logs`, 런타임 생성 파일은 제외하고 소스/문서 파일만 포함하는 것을 권장한다.
