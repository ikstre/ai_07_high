### 1-2. 광고 문구 UI 병합 + 선택 (이슈 #2)

**현황**: `streamlit_app.py:1093-1117` 에 4개 버튼 분리 (광고 문구 생성 / 한글 모델 비교 / 실사 이미지 작업 / 포스터 생성). `copy_result` 는 단일 결과(1257-1264), `copy_experiment_result` 는 별도 expander(1267-1278).

**목표**: "광고 문구 생성" 클릭 시 `/ai/copy/experiment` 호출 → 멀티 provider 결과를 카드 grid로 표시 → 사용자가 1개 선택 → `copy_result` 승격 → 이후 포스터/이미지에 반영.

**작업 순서**:
1. "광고 문구 생성" / "한글 모델 비교" 2개 버튼을 단일 버튼으로 통합
2. 내부 호출: `/ai/copy/experiment` (백엔드 이미 지원)
3. 결과 N개를 `st.columns()` 카드 grid로 렌더링, 각 카드에 "이 문구 사용" 버튼
4. 선택 시 `st.session_state["copy_result"]` 에 저장, UI에 선택 확인 표시
5. 기존 `copy_result` / `copy_experiment_result` 별도 섹션 병합

**주의**: HyperCLOVA 연결(1-1) 전이라도 qwen2.5:7b 단일 결과로 UI 구조 먼저 구현 가능.

## 2. P1 — UX 개선

### 2-1. 이미지 작업 자동 폴링 + 포스터 흐름 연결 (이슈 #3)

**현황**: 이미지 작업 제출 후 결과 확인이 수동. 포스터 생성 버튼이 이미지 완료 여부와 무관하게 활성화.

**목표**: 이미지 작업 제출 → polling loop → 완료 시 자동으로 포스터 흐름 활성화.

**구현 방향**:
- `st.empty()` + `time.sleep(3)` 루프 (혹은 `st.fragment` + auto-refresh) 로 `GET /ai/image/{job_id}` 폴링
- 완료 상태(`status == "done"`) 감지 시 이미지 URL 표시 + 포스터 생성 버튼 활성화
- 실패 상태 처리 (timeout, ComfyUI 오류)

### 3-2. API 응답 캐싱 (이슈 #9)

| 엔드포인트 | 캐싱 전략 |
|---|---|
| `GET /ai/providers` | TTL 60s in-memory (매번 파일 읽기 → `functools.lru_cache` 또는 `cachetools.TTLCache`) |
