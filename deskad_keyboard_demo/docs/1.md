# 이미지 생성 기능 QA 보고서

- 날짜: 2026-06-11
- 작성자: 김준혁
- 대상 기능: 이미지 생성 및 광고 포스터 생성
- 확인 범위: 기존 생성 결과물, 포스터 타입별 결과, 사용자 화면 기준 QA

## 1. QA 요약

현재 이미지 생성 기능은 키보드 제품과 데스크테리어 분위기, 광고 포스터 톤을 전반적으로 잘 반영하고 있습니다. 제품 이미지 자체는 시안으로 활용 가능한 수준이며, 포스터 타입에 따라 결과 방향도 어느 정도 달라집니다.

다만 실제 서비스 결과물로 사용하기에는 텍스트 레이아웃 안정성이 가장 큰 개선 포인트입니다. 특히 기능 강조형 포스터에서 SPEC 문구가 카드 영역 안에 안정적으로 정리되지 않고, 긴 문장이 잘리거나 말줄임 처리되는 문제가 확인되었습니다.

| 구분 | 확인 결과 |
|---|---|
| 이미지 톤 | 광고 포스터와 데스크테리어 분위기 반영 양호 |
| 제품 표현 | 키보드 중심 구성은 대체로 적합 |
| 주요 품질 이슈 | 기능 강조형 SPEC 문구 잘림/말줄임/카드 내부 정렬 문제 |
| 우선 개선 대상 | 텍스트 줄바꿈, 카드 영역 자동 조정, 재생성 옵션 강화 |

## 2. 항목별 QA 결과

### 2.1 정확도와 적합도

**질문 가이드**  
생성된 이미지/문구가 프롬프트의 스타일, 톤, 사용 목적을 얼마나 잘 반영하는가?



**확인 결과**

생성 결과는 키보드 제품, 책상 위 배치, 모니터 주변 데스크테리어, 고급스러운 광고 톤을 전반적으로 잘 반영합니다. 제품 중심 구도와 라이프스타일 컷 구성은 프롬프트의 의도와 잘 맞는 편입니다.

다만 광고 문구와 SPEC 정보가 함께 들어가는 포스터에서는 정보 전달 완성도가 떨어집니다. 이미지 분위기는 적합하지만, 문구가 일부 잘리거나 말줄임 처리되어 실제 광고 소재로 바로 사용하기에는 보완이 필요합니다.

**판정**

- 이미지 톤/스타일: 양호
- 제품 중심성: 양호
- 문구/정보 전달: 개선 필요

---

### 2.2 일관성과 안정성

**질문 가이드**  
같은 조건과 입력에 대해 생성된 콘텐츠들이 주제, 스타일, 톤에서 일관성을 유지하는가?



**확인 결과**

동일한 제품명과 유사한 조건에서 차분한 데스크테리어 톤과 광고 포스터 방향성은 비교적 유지됩니다. 전체 분위기는 크게 흔들리지 않습니다.

하지만 세부 레이아웃은 결과마다 차이가 있습니다. 제목 줄바꿈, CTA 위치, 제품 설명 문구 배치, SPEC 카드 내부 정렬이 일정하지 않아 같은 옵션을 사용해도 결과 품질을 예측하기 어렵습니다.

**판정**

- 전체 톤 일관성: 보통 이상
- 레이아웃 안정성: 개선 필요
- 텍스트 배치 일관성: 개선 필요

---

### 2.3 응답 속도와 부하 처리

**질문 가이드**  
콘텐츠 생성에 걸리는 시간이 실제 서비스로서 만족스러운가?



**확인 결과**

사용자 입장에서는 이미지 생성이 즉시 완료되는 기능은 아니기 때문에, 생성 중 상태 표시가 중요합니다. 현재 결과 생성 흐름은 사용할 수 있지만, 대기 중인 상태나 완료까지의 체감 시간을 더 명확하게 보여줄 필요가 있습니다.

실제 서비스 관점에서는 생성 중 안내 문구, 진행 상태, 완료 후 결과 위치 이동이 더 자연스럽게 제공되면 사용자가 기다리는 동안 불안감이 줄어들 것으로 보입니다.

**판정**

- 생성 완료 흐름: 사용 가능
- 대기 상태 안내: 개선 필요
- 완료 후 결과 인지: 개선 필요

---

### 2.4 유저 중심 인터페이스

**질문 가이드**  
유저가 자연스럽게 콘텐츠를 생성할 수 있도록 입력/결과 흐름이 설계되었는가?


**확인 결과**

입력 후 결과 포스터를 확인하는 기본 흐름은 이해하기 쉽습니다. 사용자가 제품명, 스타일, 포스터 타입을 선택하고 결과를 확인하는 흐름은 자연스럽습니다.

다만 결과물에 문제가 있을 때 다음 행동이 명확하지 않습니다. 예를 들어 SPEC 문구가 잘리거나 카드 안에서 정리되지 않았을 때 사용자가 바로 "문구 짧게 재생성", "줄바꿈 적용", "다른 템플릿으로 재생성" 같은 조치를 취하기 어렵습니다.

**판정**

- 기본 생성 흐름: 양호
- 결과 문제 대응 흐름: 개선 필요
- 사용자 재시도/수정 유도: 개선 필요

---

### 2.5 서비스 완성도

**질문 가이드**  
전체 생성 흐름이 자연스럽고, 실제 활용 가능한 수준인가?


**확인 결과**

이미지 자체의 완성도는 광고 시안으로 활용 가능한 수준까지 올라와 있습니다. 하지만 최종 결과물로 사용하기에는 텍스트 레이아웃 안정성이 가장 큰 문제입니다.

특히 기능 강조형 포스터는 SPEC 정보 전달이 핵심인데, 긴 문구가 카드 안에서 자연스럽게 줄바꿈되지 않고 말줄임 처리되는 현상이 확인됩니다. 이 문제는 실제 광고/발표/공유용 산출물에서는 품질 저하로 보일 수 있습니다.

**판정**

- 이미지 품질: 보통 이상
- 포스터 산출물 완성도: 개선 필요
- 기능 강조형 실사용성: 개선 필요

---

### 2.6 제어 가능성

**질문 가이드**  
사용자가 입력이나 옵션을 조정함으로써 원하는 방향으로 결과를 유도할 수 있는가?


**확인 결과**

포스터 타입이나 스타일 옵션을 변경하면 결과 방향은 어느 정도 달라집니다. 예를 들어 프로모션형, 기능 강조형, 제품 컷 중심 구성 등 큰 틀의 변화는 확인됩니다.

하지만 세부 결과를 안정적으로 제어하기는 어렵습니다. 특히 기능 강조형에서 SPEC 항목이 길어졌을 때 카드 안에 어떻게 배치될지 예측하기 어렵고, 사용자가 텍스트 길이/줄바꿈/카드 크기를 직접 제어할 수 없습니다.

**판정**

- 큰 방향 제어: 가능
- 세부 레이아웃 제어: 부족
- 텍스트 영역 제어: 개선 필요

## 3. 발견된 주요 이슈

| 우선순위 | 이슈 | 영향 |
|---|---|---|
| 높음 | 기능 강조형 SPEC 문구가 카드 영역 안에 안정적으로 배치되지 않음 | 실제 광고 결과물 품질 저하 |
| 높음 | 긴 문구가 줄바꿈 대신 말줄임 처리됨 | 정보 전달력 저하 |
| 중간 | 생성 중 대기 상태가 충분히 명확하지 않음 | 사용자가 진행 상황을 파악하기 어려움 |
| 중간 | 결과물 품질 문제가 있을 때 재생성/수정 흐름이 부족함 | 사용자 조작성이 낮아짐 |
| 중간 | 같은 옵션에서도 레이아웃 결과가 일정하지 않음 | 결과 품질 예측이 어려움 |

## 4. 개선 제안

1. 기능 강조형 포스터의 SPEC 카드에 자동 줄바꿈 로직을 적용합니다.
2. SPEC 문구 길이에 따라 폰트 크기, 줄 간격, 카드 높이를 자동 조정합니다.
3. 말줄임 처리 기준을 줄이고, 핵심 정보가 잘리지 않도록 정보 우선순위를 정합니다.
4. 생성 중 상태 표시를 강화하고 사용자가 기다리는 상황을 명확하게 안내합니다.
5. 결과 화면에서 "같은 조건으로 다시 생성", "문구 짧게 재생성", "다른 템플릿 적용" 버튼을 제공합니다.
6. 포스터 타입별로 텍스트 영역의 최대 길이와 카드 내부 여백을 다르게 적용합니다.

## 5. 종합 결론

현재 이미지 생성 기능은 광고 포스터의 분위기와 제품 중심 이미지를 만드는 데는 충분히 가능성이 있습니다. 다만 실제 서비스 완성도 기준에서는 결과물의 텍스트 안정성이 가장 큰 리스크입니다.

특히 기능 강조형 포스터는 SPEC 정보 전달이 핵심이므로, 텍스트 줄바꿈과 카드 내부 레이아웃 안정화가 우선적으로 필요합니다. 이 부분이 개선되면 생성 이미지의 활용도와 서비스 신뢰도가 크게 올라갈 것으로 보입니다.
